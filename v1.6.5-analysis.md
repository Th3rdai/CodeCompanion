## 2. Architecture

### Chat request lifecycle (condensed path)

```
App.jsx handleSend()
  → POST /api/chat { model, mode, messages, images?, conversationId }
  → routes/chat.js: SYSTEM_PROMPTS[mode] lookup
  → parallel: resolveAutoModel() + buildMemoryContext()
  → getCachedProjectPrompt(chatFolder || projectFolder)  ← 60s cache, 3-level tree
  → getToolsPromptAndFlags()  ← MCP + builtin tool list injected here
  → SSE headers flushed
  → chatStream() / chatComplete() → lib/ollama-client.js
  → NDJSON read loop → sendEvent({ token })
  → client RAF-batches tokens into setMessages()
```

**Abort chain:** `chatAbortRef.abort()` → fetch drops → `req.on("close")` → `chatAbortController.abort()` → Ollama connection dropped + tool-call loop exits.

### Agent tool pipeline

```
parseToolCalls(responseText)
  → 4 format fallbacks: TOOL_CALL:, XML, MiniMax JSON, MiniMax param
segmentToolCalls()  ← only if toolExec.parallel=true (default: off)
  → parallel batch (read_*/view_*/list_*) | serial (write_*/run_*/create_*)
executeTool(serverId, toolName, args)
  → serverId="builtin" → executeBuiltinTool()
  → otherwise → mcpClient.callTool() [timeout: 120s; image tools: 180s; retries: 1]
results injected as loopMessages → next round (max 25 rounds)
```

### 17 modes

All modes share `POST /api/chat`. `review`/`pentest`/`validate`/`score`/`build` have dedicated structured routes for their primary flows. Mode is sent as a body field and used only for `SYSTEM_PROMPTS[mode]` lookup.

| Mode                                              | Dedicated route            | Main component           |
| ------------------------------------------------- | -------------------------- | ------------------------ |
| chat/explain/bugs/refactor/translate-\* / diagram | —                          | inline App.jsx           |
| pentest                                           | `POST /api/pentest`        | SecurityPanel            |
| validate                                          | `POST /api/validate`       | ValidatePanel            |
| review                                            | `POST /api/review`         | ReviewPanel              |
| prompting/skillz/agentic/planner                  | `POST /api/score`          | builders/                |
| create                                            | `POST /api/create-project` | CreateWizard             |
| build                                             | `POST /api/build/*`        | BuildPanel + BuildWizard |
| terminal                                          | Electron IPC (node-pty)    | TerminalPanel            |

### Security model

- **Network gate:** `requireLocalOrApiKey` — IP=loopback or `X-CC-API-Key` matches `CC_API_SECRET`. Applied to: MCP management, `/api/files/save`, `/api/files/upload`, `/api/config POST`, agent terminal test. **Not** applied to: `/api/chat`, `/api/review`, `/api/pentest` (rate-limited instead), or any GET routes.
- **Path traversal:** `assertResolvedPathUnderAllowedRoots` + `isUnderRoot` chain for folder operations. `validateCwd` / `validateProjectFilePath` use `fs.realpathSync` to resolve symlinks before boundary check.
- **projectFolder vs chatFolder:** `chatFolder ⊂ projectFolder` enforced server-side in `routes/config.js`. `chatFolder` scopes AI context and file browser; `projectFolder` is the write/terminal security boundary.
- **Terminal:** allowlist → blocklist → metachar scan → `validateCwd` (realpathSync) → whitelisted env only → optional `confirmBeforeRun`.

### Auto-model resolution (`lib/auto-model.js`)

```
model="auto" → mergeAutoModelMap(config) → mode default → listModels(45s cache)
→ vision override (largest local vision model <15GB)
→ tool-capable override (if agentTerminal.enabled and cloud model requested)
→ matchAvailable() with :latest suffix tolerance
→ resolvedModel SSE event sent before first token
```

Default map: `chat→gemma4`, `review/pentest/diagram→qwen3-32k`, `explain/bugs/validate/create/build→kimi-k2:1t-cloud`.

### File/document pipeline

```
GET /api/files/tree → buildFileTree(chatFolder, depth≤6, 5000-node cap)
                    → assertResolvedPathUnderAllowedRoots
                    → cached 60s in getCachedProjectPrompt

POST /api/convert-document
  → Docling first (if enabled, port 5002)
  → built-in fallback (pdf-parse/mammoth/read-excel-file/officeparser)
  → 503 for unsupported without Docling
```

### Streaming

SSE format: `data: {...}\n\n` for tokens, `data: [DONE]\n\n` terminator. Uses `fetch` + `ReadableStream` (not `EventSource`) to allow POST with AbortController. Context window auto-adjusted: `effectiveNumCtx = min(chars/3.5 + 2048, 524288)`; timeout scaled +60s per additional 32K estimated tokens.

## 4. Concerns & Risks

### Security findings

#### S2 — Shell injection in `lib/github.js` (HIGH)

`lib/github.js:671`:

```js
execSync(`git checkout -b ${branch}`, { cwd: localPath });
execSync(`git push -u origin ${branch}`, { cwd: localPath });
```

`branch` comes from `req.body.branch` with **no sanitization**. Value like `main; rm -rf /` executes arbitrary commands. Route is protected by `requireLocalOrApiKey` but a stored-XSS or SSRF that triggers the endpoint could still execute.

**Fix:** Use `execFileSync(['git', 'checkout', '-b', branch])` (args array, no shell interpolation). Validate `branch` against `/^[a-zA-Z0-9_.\-\/]+$/`.

#### S1 — GET routes unauthenticated on LAN (MEDIUM)

`GET /api/config`, `GET /api/files/tree`, `GET /api/files/read`, `GET /api/files/read-raw` have no `requireLocalOrApiKey` guard. When `CC_BIND_ALL=1`, any LAN host can read project file tree and arbitrary files within allowed roots, plus the full (partially sanitized) config.

**Fix:** Add `requireLocalOrApiKey` to all four GET routes.

#### S4 — `POST /api/files/upload` unauthenticated, no size limit (MEDIUM)

`routes/files.js:250–256` — no auth guard, no explicit body size limit beyond Express default. Echoes content back; doesn't write to disk directly, but is an unprotected endpoint accepting arbitrary payloads when on LAN.

#### S3 — `electron/ide-launcher.js` shell injection via `folder` (LOW-MEDIUM)

`launch-ide` IPC passes unvalidated `folder` to shell commands via template literals. Linux path uses `x-terminal-emulator -e "cd '${folder}' && ..."`. Requires compromised renderer to exploit (contextIsolation is on).

**Fix:** Validate `folder` is within project root before passing to `launchIDE`. Switch `exec` to `execFile` with args array.

### Performance

#### App.jsx monolith (65 hooks, 2,233 lines)

Every state change risks cascading re-renders through the full component tree. No context splitting, no reducer. **Fix:** Extract `useBuilderState`, `useBuildDashboard`, `useMcpConfig` custom hooks.

#### History list — synchronous full scan

`listConversations()` reads every history file synchronously on every sidebar open. No index, no pagination. Legacy `resolveConversationFilePath` has O(N) scan fallback. **Fix:** Lightweight `history-index.json` with `{id,title,mode,createdAt}`; full files loaded on demand.

#### File tree walker — sync recursive I/O

`buildFileTree` + `readFolderFiles` block the event loop. Existing caps (5,000 nodes, 1,500ms polling) are sufficient for typical projects but not for large monorepos. **Fix:** async `fs.promises.readdir` walking.

#### MCP tool injection — unbounded system prompt growth

159 tools currently connected (Archon 17, Google 121, crawl4ai-rag 5, Stitch 12, nano banana 4). Estimated 3,000–5,000 tokens injected per request. For 8K-context local models, this leaves ~3K–5K tokens for conversation history. **Fix:** Per-server tool cap configurable in MCP Clients settings; surface injected token count in debug log.

#### Memory `_persistToDisk` — synchronous write on every add

With large embedding vectors (nomic-embed-text = ~4K floats per entry) at 500 memories = ~8MB JSON, `writeFileSync` on every `addMemory` call blocks the event loop. **Fix:** 500ms debounce on `_persistToDisk`.

### Scalability

#### Config — non-atomic writes

`lib/config.js:165–167` uses bare `writeFileSync` (no tmp→rename). `electron/main.js` reads the same file directly. Concurrent writes from main + server child processes can produce torn JSON. Contrast: history and memory both use atomic rename correctly. **Fix:** Write to `.tmp` then `renameSync` in `saveConfig`.

#### Memory — deactivated entries never pruned

500-entry cap applies only to `active` memories. `active: false` entries accumulate indefinitely, growing `memories.json` without bound. No API to compact. **Fix:** Add `POST /api/memory/compact` that rewrites to active-only entries; or prune deactivated entries >30 days old in `_autoPrune`.

### Electron-specific

#### `terminal-write`/`terminal-resize` — no input validation

`terminal-write` passes `data` directly to `pty.write()` with no type check or size limit. `terminal-resize` passes `cols`/`rows` unvalidated — `cols=0` or `cols=99999` can crash `node-pty`. **Fix:** typeof + 64KB max on write; clamp cols/rows to `[1, 500]`.

#### `terminal-start` — config read TOCTOU

`electron/main.js:947–954` reads `.cc-config.json` synchronously inside the IPC handler. If `saveConfig` is writing (non-atomic) concurrently, the read can get partial JSON. **Fix:** Request config from Express loopback API instead of re-reading disk.

#### `asar: false` — source fully readable in packaged app

`lib/prompts.js` (scoring prompts), `lib/review.js`, `lib/pentest.js` are all plain-text in the installed `.app`. Known trade-off; investigate `asarUnpack` for native modules + `asar: true` for JS.

### Dependency risks

| Package                        | Patch                     | Risk if upgraded                        |
| ------------------------------ | ------------------------- | --------------------------------------- |
