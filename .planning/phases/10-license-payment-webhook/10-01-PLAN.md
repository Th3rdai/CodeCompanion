# Phase 10: License Payment Webhook — Plan

**Phase:** 10-license-payment-webhook
**Type:** execute
**Depends on:** Phase 9

## Objective

Generate and deliver license keys automatically when a customer completes a purchase. Support Stripe (primary) and optionally Paddle.

## Tasks

### Task 1: License keys storage

**File:** lib/license-db.js (new) or extend config

1. Define schema: `license_keys` (id, email, key, features, expires, order_id, created_at)
2. If no DB: use JSON file in data dir (e.g. `license-keys.json`) with append/sync
3. If DB: add SQLite or use existing store; migrations
4. `insertLicenseKey(record)` — returns id
5. `getLicenseByOrderId(order_id)` — for idempotency

### Task 2: Webhook endpoint

**File:** server.js or routes/webhooks.js

1. `POST /api/webhooks/stripe` — verify signature with `STRIPE_WEBHOOK_SECRET`
2. Parse `checkout.session.completed` (or `payment_intent.succeeded`)
3. Extract: `customer_email`, `metadata.features`, `metadata.expires`, `metadata.order_id`
4. If `metadata.expires` missing, default to 1 year from now
5. Idempotency: if `getLicenseByOrderId(order_id)` exists, return 200 (no duplicate key)
6. Call `generateKey({ email, features, expires })` from lib/license-generator
7. `insertLicenseKey({ email, key, features, expires, order_id })`
8. Trigger email send (or queue)
9. Return 200 quickly (< 5s) — do email async if needed

### Task 3: Stripe metadata configuration

**Documentation:** Stripe product/price setup

1. Ensure checkout metadata includes: `features` (e.g. `skillz,agentic`), `order_id`
2. Optional: `expires` for custom terms
3. Document in README or internal docs

### Task 4: Error handling

1. Invalid payload: log, return 400
2. Signature verification failure: return 401
3. Key generation failure: log, return 500 (Stripe will retry)
4. Email failure: log, don't fail webhook — key is stored; manual resend possible
