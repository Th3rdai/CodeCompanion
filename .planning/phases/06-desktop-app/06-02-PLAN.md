  <name>Task 1: Cross-platform IDE launcher and server.js migration</name>
  <files>electron/ide-launcher.js, server.js, electron/main.js, electron/preload.js</files>
  <action>
1. Create `electron/ide-launcher.js`:
   - Export `launchIDE(ideName, folder)` function
   - Platform switch on `process.platform` for each IDE:
     - **cursor**: darwin: `open -a "Cursor" "${folder}"`, win32: `cmd /c start "" "cursor" "${folder}"`, linux: `cursor "${folder}"`
     - **windsurf**: darwin: `open -a "Windsurf" "${folder}"`, win32: `cmd /c start "" "windsurf" "${folder}"`, linux: `windsurf "${folder}"`
     - **claude-code**: darwin: `osascript -e 'tell application "Terminal" to do script "cd \"${folder}\" && claude"'`, win32: `cmd /c start cmd /k "cd /d "${folder}" && claude"`, linux: `x-terminal-emulator -e "cd '${folder}' && claude"`
     - **opencode**: same terminal pattern as claude-code but with `opencode` command
   - Use `child_process.exec` (async, not execSync) wrapped in a Promise
   - Throw descriptive error if IDE/platform combo unsupported

2. Update `server.js` IDE launcher endpoints (lines ~573-630):
   - Keep the existing Express routes (`/api/launch-cursor`, `/api/launch-windsurf`, `/api/launch-claude`, `/api/launch-opencode`)
   - Replace the hardcoded macOS `execSync('open -a ...')` and `execSync('osascript ...')` with platform-aware logic:
     - If running in Electron (`process.env.CC_DATA_DIR` is set or check `process.versions.electron`): import and use `launchIDE()` from `electron/ide-launcher.js`
     - If running as plain Node (dev mode): keep current macOS behavior as fallback
   - This way the Express API endpoints remain the same interface, but the implementation is now cross-platform

3. Add IPC handler in `electron/main.js` for IDE launching (as a backup channel if needed):
   - `ipcMain.handle('launch-ide', (event, { ide, folder }) => launchIDE(ide, folder))`

4. Add to `electron/preload.js`:
   - `launchIDE: (ide, folder) => ipcRenderer.invoke('launch-ide', { ide, folder })`
     </action>

<task type="auto">
  <name>Task 2: Ollama setup wizard with auto-install, model pull progress, and connection health dot</name>
  <files>electron/ollama-setup.js, electron/main.js, electron/preload.js, src/components/OllamaSetup.jsx, src/components/ConnectionDot.jsx, src/App.jsx</files>
  <action>
1. Create `electron/ollama-setup.js`:
   - `checkOllamaRunning(ollamaUrl)`: fetch `${ollamaUrl}/api/tags` with 3s timeout, return `{ running: boolean, models: string[] }`
   - `installOllama()`: detect `process.platform`:
     - darwin: download `https://ollama.com/download/Ollama-darwin.zip`, extract, open the .app
     - win32: download `https://ollama.com/download/OllamaSetup.exe`, spawn installer
     - linux: spawn `curl -fsSL https://ollama.com/install.sh | sh` in a terminal
     - After launching installer, poll `http://localhost:11434` every 2s up to 120s until responsive
     - Return `{ success: boolean, error?: string }`
   - `pullModel(ollamaUrl, modelName, onProgress)`:
     - POST to `${ollamaUrl}/api/pull` with `{ name: modelName, stream: true }`
     - Parse NDJSON response stream line by line
     - Call `onProgress({ status, total, completed, percent })` for each line
     - `percent` = `completed / total * 100` when both are available
     - Return `{ success: boolean }` when status is "success"

2. Add IPC handlers in `electron/main.js`:
   - `check-ollama`: calls `checkOllamaRunning()` with configured URL
   - `install-ollama`: calls `installOllama()`, sends progress events to renderer
   - `pull-model`: calls `pullModel()`, forwards progress via `event.sender.send('pull-progress', data)`

3. Add to `electron/preload.js`:
   - `checkOllama: () => ipcRenderer.invoke('check-ollama')`
   - `installOllama: () => ipcRenderer.invoke('install-ollama')`
   - `pullModel: (modelName) => ipcRenderer.invoke('pull-model', modelName)`
   - `onPullProgress: (callback) => ipcRenderer.on('pull-progress', (_, data) => callback(data))`
   - `offPullProgress: () => ipcRenderer.removeAllListeners('pull-progress')`

4. Create `src/components/OllamaSetup.jsx`:
   - A friendly setup wizard component (NOT an error state per user decision)
   - Three states:
     a. **Not Connected**: Friendly illustration/icon, "Let's connect to Ollama" heading, explanation of what Ollama is ("It's the AI engine that powers Code Companion — it runs on your computer so your code stays private"), two buttons: "Install Ollama for me" (calls electronAPI.installOllama) and "I already have it — check again" (re-checks). Also a "Configure URL" link to expand an input for custom Ollama URL.
     b. **Installing**: Progress screen "Installing Ollama..." with a spinner/progress indicator, status text updates
     c. **No Models**: "Ollama is running! Now let's get a model." Recommend `qwen3:latest` with explanation ("A smart, fast model that works great for code review"). "Pull qwen3:latest" button. Progress bar showing download percentage and size (e.g., "Downloading... 2.1 GB / 4.7 GB (45%)"). After success: "You're all set!" with a "Start using Code Companion" button.
   - Use Lucide icons (Wifi, WifiOff, Download, CheckCircle, AlertCircle) — no emoji icons per skill rules
   - Tailwind styling consistent with existing app design

5. Create `src/components/ConnectionDot.jsx`:

   - Include tooltip text: "Connected to Ollama" / "Ollama disconnected"
