3. Create `electron/main.js` — the Electron main process:
   - Import electron `app`, `BrowserWindow`, `shell`, `dialog`
   - Import `child_process.fork`
   - `findFreePort(preferredPort)`: use `net.createServer()` — try preferred port first, fall back to port 0 for random free port. Return `{ port, wasFallback, preferredPort }`.
   - `startApp()` flow:
     a. Call `resolveDataDirectory()` from data-manager.js
     b. Read preferred port from config (default 3000), validate range 1024-65535
     c. Call `findFreePort(preferredPort)` to get actual port
     d. **Spawn server as child process per user decision (NOT require()):**
        ```javascript
        const serverProcess = fork(path.join(__dirname, '..', 'server.js'), [], {
          env: {
            ...process.env,
            PORT: String(actualPort),
            CC_DATA_DIR: dataDir
          },
          stdio: ['pipe', 'pipe', 'pipe', 'ipc']
        });
        ```
     e. Listen for IPC ready message from server child:
        ```javascript
        serverProcess.on('message', (msg) => {
          if (msg.type === 'server-ready') {
            // Server is listening, navigate main window to localhost:PORT
          }
        });
        ```
     f. Create `BrowserWindow` with:
        - `title: 'Code Companion — Vibe Coder Edition'` per user decision
        - `minWidth: 1024, minHeight: 768` per user decision
        - `backgroundColor: '#0f172a'` (Tailwind slate-900) per user decision to prevent flash of white
        - `show: false` (show after ready-to-show)
        - `webPreferences: { preload: path.join(__dirname, 'preload.js'), contextIsolation: true, nodeIntegration: false }`
        - Load window state from `window-state.js` for x/y/width/height
     g. **Load splash screen first:** `win.loadFile(path.join(__dirname, 'splash.html'))` — show splash while server boots
     h. On `ready-to-show`: if saved state was maximized, `win.maximize()`, then `win.show()`
     i. Once server child sends ready message: `win.loadURL('http://localhost:' + actualPort)` — navigates away from splash to the app
     j. If port was a fallback (wasFallback=true), send IPC message to renderer so it can show a toast: `win.webContents.send('port-fallback', { actual: actualPort, preferred: preferredPort })`
     k. On `close`: save window state via `saveWindowState(win, dataDir)`
     l. Set application menu from `menu.js`
   - **Server crash handling per user decision:**
     ```javascript
     serverProcess.on('exit', (code, signal) => {
       if (code !== 0 && signal !== 'SIGTERM') {
         dialog.showMessageBox(win, {
           type: 'error',
           title: 'Server Stopped',
           message: 'The server stopped unexpectedly. Your data is safe.',
           buttons: ['View Logs', 'Restart Server', 'Quit'],
           defaultId: 1
         }).then(({ response }) => {
           if (response === 0) shell.openPath(logDir);
           else if (response === 1) respawnServer();
           else app.quit();
         });
       }
     });
     ```
   - **Graceful shutdown per user decision:**
     ```javascript
     app.on('before-quit', () => {
       if (serverProcess && !serverProcess.killed) {
         serverProcess.kill('SIGTERM');
         // Wait up to 5 seconds, then force kill
         setTimeout(() => {
           if (serverProcess && !serverProcess.killed) {
             serverProcess.kill('SIGKILL');
           }
         }, 5000);
       }
     });
     ```
   - `app.on('window-all-closed', () => app.quit())` — close button quits per user decision
   - `app.whenReady().then(startApp)`
   - Register IPC handlers:
     - `get-app-version`: returns `app.getVersion()`
     - `get-data-dir`: returns resolved data directory path
     - `export-data`: calls `exportData()` from data-manager.js, opens native save dialog
     - `import-data`: calls `importData()` from data-manager.js, opens native file dialog
     - `get-last-mode`: reads from config
     - `set-last-mode`: writes to config
     - `get-port-config`: reads preferred port from config
     - `set-port-config`: writes preferred port to config (validated 1024-65535)
     - `get-actual-port`: returns the actual port the server is running on
