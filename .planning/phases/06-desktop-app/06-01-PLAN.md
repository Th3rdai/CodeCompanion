<task type="auto">
  <name>Task 1: Install Electron dependencies and create main process with embedded Express</name>
  <files>package.json, electron/main.js, electron/preload.js, electron/menu.js, electron/window-state.js, electron/data-manager.js, resources/data-readme.txt, server.js</files>
  <action>
1. Install dependencies:
   ```
   npm install --save-dev electron electron-builder
   npm install electron-updater electron-log archiver extract-zip
   ```

2. Add scripts to package.json:
   ```json
   "electron:dev": "electron electron/main.js",
   "electron:build": "npm run build && electron-builder"
   ```
   Also add `"main": "electron/main.js"` (change from `"main": "server.js"`). Keep the existing `"start": "node server.js"` for non-Electron use.

3. Create `electron/main.js` — the Electron main process:
   - Import electron `app`, `BrowserWindow`, `shell`
   - `findFreePort()`: use `net.createServer()` with port 0 to detect a free port
   - `startApp()` flow:
     a. Call `resolveDataDirectory()` from data-manager.js
     b. Set `process.env.CC_DATA_DIR` to resolved data dir
     c. Set `process.env.PORT` to the free port
     d. `require('../server.js')` to start Express in-process
     e. Create `BrowserWindow` with:
        - `show: false` (show after ready-to-show)
        - `webPreferences: { preload: path.join(__dirname, 'preload.js'), contextIsolation: true, nodeIntegration: false }`
        - Load window state from `window-state.js` for x/y/width/height
     f. `win.loadURL('http://localhost:' + port)`
     g. On `ready-to-show`: if saved state was maximized, `win.maximize()`, then `win.show()`
     h. On `close`: save window state via `saveWindowState(win, dataDir)`
     i. Set application menu from `menu.js`
   - `app.on('window-all-closed', () => app.quit())` — close button quits per user decision
   - `app.whenReady().then(startApp)`
   - Register IPC handlers:
     - `get-app-version`: returns `app.getVersion()`
     - `get-data-dir`: returns resolved data directory path
     - `export-data`: calls `exportData()` from data-manager.js, opens native save dialog
     - `import-data`: calls `importData()` from data-manager.js, opens native file dialog
     - `get-last-mode`: reads from config
     - `set-last-mode`: writes to config

4. Create `electron/preload.js`:
   - Use `contextBridge.exposeInMainWorld('electronAPI', {...})` to expose:
     - `getAppVersion()` -> invoke 'get-app-version'
     - `getDataDir()` -> invoke 'get-data-dir'
     - `exportData()` -> invoke 'export-data'
     - `importData()` -> invoke 'import-data'
     - `getLastMode()` -> invoke 'get-last-mode'
     - `setLastMode(mode)` -> invoke 'set-last-mode'
     - `isElectron: true`

5. Create `electron/menu.js`:
   - Export a function `createMenu()` that returns a Menu template
   - Include: Edit (Undo, Redo, Cut, Copy, Paste, Select All), View (Reload, Force Reload, Toggle DevTools, separator, Actual Size, Zoom In, Zoom Out, Full Screen), Window (Minimize, Close), Help (Learn More -> opens GitHub repo URL in browser)
   - No File menu per user decision (app is self-contained)

6. Create `electron/window-state.js`:
   - `loadWindowState(dataDir)`: reads `window-state.json` from dataDir, returns defaults `{ width: 1200, height: 800, isMaximized: true }` if file missing
   - `saveWindowState(win, dataDir)`: saves `{ x, y, width, height, isMaximized }` to `window-state.json`

7. Create `electron/data-manager.js`:
   - `resolveDataDirectory()`:
     - On macOS/Linux: try `path.join(path.dirname(process.execPath), 'CodeCompanion-Data')` first
     - Write-test: create `.write-test` file, delete it
     - If fails (code signing): fall back to `app.getPath('userData')`
     - On Windows: use `path.join(app.getPath('appData'), 'CodeCompanion-Data')` as primary (Program Files is read-only per research open question #4)
     - Create directory if not exists, copy `resources/data-readme.txt` as `README.txt` inside
   - `migrateDevData(dataDir)`:
     - Check for `./data/` relative to app root (or `__dirname + '/../data/'`)
     - If exists and data dir is empty: copy all files from `./data/` to `dataDir`
     - Also check for `./history/` and `./config.json` in the dev location
     - Log migration results
   - `createBackup(dataDir)`:
     - Use `archiver` to create `CodeCompanion-Backup-YYYY-MM-DD.zip`
     - Exclude other backup ZIPs from the archive
     - Return backup file path
   - `exportData(dataDir)`:
     - Create ZIP of entire data directory
     - Return the ZIP buffer or path for the save dialog
   - `importData(dataDir, zipPath)`:
     - Use `extract-zip` to extract ZIP contents into data directory
     - Merge, don't overwrite existing files (or overwrite — user's choice via dialog)

8. Create `resources/data-readme.txt`:
   ```
   Code Companion Data Directory
   =============================
   This folder contains your Code Companion data:
   - config.json — Your settings and preferences
   - history/ — Your conversation and review history
   - logs/ — Application logs

   DO NOT DELETE this folder unless you want to reset Code Companion.
   Your data persists across app updates.

   To back up: Use Settings > Export Data in the app.
   To restore: Use Settings > Import Data in the app.
   ```

9. Modify `server.js` minimally:
   - Line ~101: Change `initConfig(__dirname)` to `initConfig(process.env.CC_DATA_DIR || __dirname)`
   - Same for `initHistory` and `createLogger`
   - Line ~935: Change `app.listen(PORT, () => {` to export a function or just let it continue working (the listen call is fine — Electron sets PORT before requiring the file)
   - Add `module.exports = app;` at the end so Electron can reference it if needed
