# Phase 9: License Batch Generation — Plan

**Phase:** 09-license-batch-generation
**Type:** execute
**Depends on:** Phase 7

## Objective

Enable bulk license key generation for hundreds or thousands of keys. Extract shared logic so future payment webhooks and APIs can reuse it.

## Tasks

### Task 1: Extract shared license generator

**File:** lib/license-generator.js (new)

1. Create module that exports `generateKey({ email, tier?, features?, expires })`
2. Read private key from `scripts/.license-private-key` or configurable path
3. Build payload: `{ email, exp, nonce }` + tier or features
4. Sign with Ed25519, return `CC-PRO-${payloadB64}.${signatureB64}`
5. Validate inputs (email, expires, tier/features) — same rules as current script
6. Export `VALID_FEATURES` constant

### Task 2: Refactor generate-license-key.js

**File:** scripts/generate-license-key.js

1. Replace inline signing logic with `require('./lib/license-generator').generateKey`
2. Keep CLI parsing, help, generateKeypair
3. Single-key path: call `generateKey(args)`, log output

### Task 3: Add batch mode

**File:** scripts/generate-license-key.js

1. Add `--batch <file>` and `--format csv|json`
2. Input CSV: `email,features,expires` (header row optional)
   - features: comma or semicolon separated (e.g. `skillz,agentic`)
   - expires: YYYY-MM-DD
3. Input JSON: array of `{ email, features?, tier?, expires }`
4. Output: CSV or JSON with `email,licenseKey,expires`
5. Optional: `--output <file>` to write to file instead of stdout
6. Progress: log every N keys (e.g. 100)
7. On row error: log error, skip row, continue (or `--strict` to abort)

### Task 4: Documentation

**File:** scripts/README-license.md (new)

1. Document single-key usage
2. Document batch usage with examples
3. Security: private key location, never commit
4. Example CSV/JSON formats
