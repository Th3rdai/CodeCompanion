# Phase 12: License Key Pool — Plan

**Phase:** 12-license-key-pool
**Type:** execute
**Depends on:** Phase 9

## Objective

Pre-generate a pool of license keys for manual or claim-based distribution.

## Tasks

### Task 1: Pool generation mode

**File:** scripts/generate-license-key.js

1. Add `--pool N` — generate N keys
2. Required: `--features` or `--tier`, `--expires`
3. Optional: `--email-placeholder` for payload (e.g. `pool@th3rdai.com`) or omit email
4. Output: CSV/JSON with `key,features,expires,claimed` (false)
5. Write to `--output <file>` (required for pool)
6. Progress logging every 1000 keys

### Task 2: Pool storage

**File:** lib/license-pool.js (new)

1. Load pool from JSON/CSV
2. Schema: `{ key, features, expires, claimed_at?, order_id?, email? }`
3. `claimNextKey(email, orderId)` — returns first unclaimed key, marks claimed
4. `getPoolStats()` — total, claimed, remaining

### Task 3: Optional claim API

**File:** server.js or routes/license-api.js

1. `POST /api/license/claim` — body: `{ email, orderId }`
2. Requires payment verification (e.g. orderId from payment provider)
3. Calls `claimNextKey`, returns key
4. Idempotency: same orderId returns same key
5. Rate limit and auth (e.g. internal service only)

### Task 4: Security documentation

1. Pool file must be protected (not in repo, restricted access)
2. Rotate unused keys periodically if exposed
3. Document when to use pool vs webhook generation

## Verification

- [ ] `--pool 100` produces 100 valid keys
- [ ] claimNextKey returns unique keys, marks claimed
- [ ] Same orderId returns same key (idempotent)
