# Phase 14: License Revocation — Plan

**Phase:** 14-license-revocation
**Type:** execute
**Depends on:** Phase 10 or 11

## Objective

Support revoking license keys. Online validation returns invalid for revoked keys.

## Tasks

### Task 1: Revocation storage

**File:** lib/license-db.js or lib/license-revocation.js

1. Table/list: `revoked_licenses` (key_hash, revoked_at, reason, revoked_by)
2. key_hash: SHA256 of full key (don't store raw key)
3. `revoke(keyHash | key)` — add to list
4. `isRevoked(key)` — hash key, check list
5. Index for fast lookup

### Task 2: Validation integration

**File:** lib/license-manager.js or validation endpoint

1. After signature verification, call `isRevoked(key)`
2. If revoked: return `{ valid: false, reason: 'revoked' }`
3. Offline validation: no revocation check (key still works until next online check)

### Task 3: Admin revoke

**Options:** CLI or API

1. **CLI:** `node scripts/revoke-license.js --key <key>` or `--email x@y.com`
2. **API:** `POST /api/admin/license/revoke` (Phase 11)
3. Lookup key by email/orderId if needed, compute hash, add to revoked list
4. Require admin auth

### Task 4: Audit log

1. Log: timestamp, key_hash (or email), reason, admin_id
2. Optional: export for compliance

### Task 5: Refund webhook extension (optional)

**File:** webhook handler

1. On Stripe `charge.refunded`, find order_id, revoke associated key
2. Idempotent: already revoked is no-op

## Verification

- [ ] Revoked key returns valid: false from validation API
- [ ] Non-revoked key still valid
- [ ] Revoke by email finds and revokes correct key
- [ ] Audit log contains revoke events
