  <name>Task 1: POST /api/build/projects/:id/research and /plan SSE endpoints with write-after-validate</name>
  <files>server.js</files>
  <action>
Add two new SSE endpoints in server.js after the existing next-action route. Follow the SSE pattern from RESEARCH.md Pattern 2.

**POST /api/build/projects/:id/research:**
- Resolve project via `_resolveBuildProject`
- Extract `model` and `phaseNumber` from `req.body`
- Create GsdBridge, get state and roadmap
- Set SSE headers (Content-Type, Cache-Control, Connection, flushHeaders)
- Build system prompt: "You are a technical researcher preparing context for project planning. Given the project state and roadmap, research phase {phaseNumber}. Identify: 1) What needs to be built, 2) Key technical decisions, 3) Dependencies and risks, 4) Suggested approach. Be thorough but concise. Use markdown formatting."
- User message: include state (truncated to 3000 chars) and roadmap phases (truncated to 3000 chars)
- Call `chatComplete(config.ollamaUrl, model, messages, 180000)` — use 3min timeout for research
- Stream result progressively: split into words, send each as `{ token: word }` via sendEvent
- Send `{ done: true }` at end
- On error: `sendEvent({ error: err.message })`
- Add `req.on('close', ...)` handler to set an abort flag, check before sendEvent calls
- End with `res.end()`

**POST /api/build/projects/:id/plan:**
- Same pattern as research but different prompt
- System prompt: "You are a project planner. Given the research context and project state, create a concrete plan for phase {phaseNumber}. Include: 1) Phase goal, 2) Tasks with specific file paths and actions, 3) Success criteria, 4) Estimated complexity. Format as markdown with clear headings."
- Extract `researchContext` from `req.body` (the research output from previous step) and include it in user message
- After getting result, apply write-after-validate: check non-empty, check starts with `#`, check length > 100 chars
- If validation passes and `req.body.writeToFile` is true, write to `.planning/phases/` directory using atomic write (write to `.tmp.${process.pid}` then rename). Use `isWithinBasePath` from `lib/file-browser.js` to validate the path.
- Send validation result in final event: `{ done: true, validated: true/false, written: true/false }`

**Rate limiting:** Add rate limiters for both endpoints:
```javascript
app.use('/api/build/projects/:id/research', createRateLimiter({ name: 'build-research', max: 5, windowMs: 60000, methods: ['POST'] }));
app.use('/api/build/projects/:id/plan', createRateLimiter({ name: 'build-plan', max: 5, windowMs: 60000, methods: ['POST'] }));
```
