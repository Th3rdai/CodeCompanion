**Backend (server.js):**
Add `POST /api/build/projects/:id/next-action` after the existing `/phase/:n` route (around line 1141). Follow the exact pattern from RESEARCH.md Pattern 4:
- Resolve project via `_resolveBuildProject(req, res)`
- Extract `model` from `req.body` (use `getConfig().selectedModel` as fallback)
- Create a GsdBridge, call `getState()` and `getProgress()`
- Build messages array with system prompt: "You are a friendly project coach. Given the project state, suggest the single most important next action. Be concise (2-3 sentences). Use encouraging, non-technical language. If the project is complete, congratulate them."
- User message: include stringified state and progress (truncate state to first 2000 chars to avoid overwhelming small models)
- Call `chatComplete(config.ollamaUrl, model, messages, 30000)` with 30s timeout
- Return `{ action: result, timestamp: new Date().toISOString() }`
- On error, return 500 with `{ error: err.message }`
- Add rate limiting: `app.use('/api/build/projects/:id/next-action', createRateLimiter({ name: 'build-next-action', max: 10, windowMs: 60000, methods: ['POST'] }))`
