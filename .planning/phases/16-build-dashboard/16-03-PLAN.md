**Backend (server.js):**
Add file endpoints after existing build routes.

Whitelist constant (add near top of build routes section around line 1030):
```javascript
const PLANNING_FILE_WHITELIST = [
  'ROADMAP.md', 'REQUIREMENTS.md', 'STATE.md', 'CONTEXT.md',
  'PROJECT.md', 'RETROSPECTIVE.md', 'config.json'
];
```

GET /api/build/projects/:id/files — list available planning files:
- Resolve project, read `.planning/` dir, filter to only PLANNING_FILE_WHITELIST entries that exist on disk
- Return `{ files: [...filenames] }`

GET /api/build/projects/:id/files/:filename:
- Resolve project, check `PLANNING_FILE_WHITELIST.includes(filename)` — 403 if not
- Build path: `path.join(project.path, '.planning', filename)`
- Validate `isWithinBasePath(path.join(project.path, '.planning'), fullPath)` — 403 if fails
- Check exists — 404 if missing
- Return `{ content: fs.readFileSync(fullPath, 'utf-8'), filename }`

PUT /api/build/projects/:id/files/:filename:
- Same whitelist + path traversal checks
- Validate `req.body.content` is non-empty string — 400 if invalid
- Atomic write: `fs.writeFileSync(fullPath + '.tmp.' + process.pid, content, 'utf-8')` then `fs.renameSync(tmpPath, fullPath)`
- Return `{ success: true, filename }`
- try/catch with 500 on failure
