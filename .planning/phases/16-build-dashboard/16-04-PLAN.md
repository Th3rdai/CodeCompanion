  <action>
Create `src/components/ClaudeCodeHandoff.jsx`.

Props: `{ project, projectData, onToast }`

The component shows a "Handoff to Claude Code" section with pre-built GSD slash commands the user can copy-paste into Claude Code or Cursor.

Structure:

- Header: "Open in Claude Code" with Lucide `Terminal` icon
- Brief explanation: "Copy these commands to continue building in Claude Code, Cursor, or Windsurf."
- Command cards — each is a glass card with:
  - Command name (bold, small text)
  - Description (text-xs text-slate-400)
  - Code block (font-mono, text-indigo-300, bg-slate-900/60 rounded-lg px-4 py-2.5)
  - Copy button (Lucide `Copy` icon, calls navigator.clipboard.writeText then onToast('Copied!'))

Commands to show (derive from project state):

1. **Open project directory:**
   - Command: `cd ${project.path}`
   - Description: "Navigate to your project folder"

2. **Start new project (if no phases):**
   - Command: `/gsd:new-project`
   - Description: "Set up your project roadmap and requirements"
   - Only show if projectData.roadmap.phases is empty or missing

3. **Plan next phase (if phases exist but incomplete):**
   - Determine next incomplete phase number from projectData.roadmap.phases
   - Command: `/gsd:plan-phase ${nextPhaseNum}`
   - Description: "Plan the next phase of development"
   - Only show if there are incomplete phases

4. **Execute phase (if plans exist):**
   - Command: `/gsd:execute-phase ${currentPhaseNum}`
   - Description: "Execute the current phase plans"
   - Only show if there are phases with plans

5. **Verify work:**
   - Command: `/gsd:verify-work ${currentPhaseNum}`
   - Description: "Verify the current phase is complete"

6. **Research phase:**
   - Command: `/gsd:research-phase ${nextPhaseNum}`
   - Description: "Deep research before planning a complex phase"

Use conditional rendering to only show relevant commands based on project state. All clickable elements get `cursor-pointer`. Use Lucide icons only (no emojis).
</action>
