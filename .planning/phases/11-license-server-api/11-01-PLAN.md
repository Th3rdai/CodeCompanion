# Phase 11: License Server API — Plan

**Phase:** 11-license-server-api
**Type:** execute
**Depends on:** Phase 9, Phase 10 (for revocation storage)

## Objective

Provide online validation and revocation so keys can be checked server-side and revoked when needed.

## Tasks

### Task 1: Validation endpoint

**File:** server.js or routes/license-api.js

1. `POST /api/license/validate` — body: `{ key }`
2. Verify signature with embedded public key (reuse license-manager validateKey)
3. If valid: check revocation table/flag (if Phase 14 done)
4. Return: `{ valid: true, email, features, expires }` or `{ valid: false, reason }`
5. Rate limit: e.g. 60 req/min per IP
6. No auth required — key itself is the credential

### Task 2: Revocation storage

**File:** lib/license-db.js

1. Table or list: `revoked_keys` (key_hash or key_id, revoked_at, reason)
2. `revokeKey(keyId | email | orderId)` — mark as revoked
3. `isRevoked(key)` — check before returning valid

### Task 3: Admin revoke endpoint

**File:** server.js or routes/admin.js

1. `POST /api/admin/license/revoke` — body: `{ keyId | email | orderId }`
2. Require `Authorization: Bearer <ADMIN_API_KEY>` or similar
3. Call `revokeKey`, return success/failure
4. Audit log: who revoked, when, what

### Task 4: Optional app-side online check

**File:** lib/license-manager.js, config

1. Config: `LICENSE_VALIDATION_MODE: 'offline' | 'online' | 'hybrid'`
2. Offline: current behavior (no server call)
3. Online: on activate, call `/api/license/validate`; cache result for N days
4. Hybrid: offline first; periodic background validate (e.g. weekly)

## Verification

- [ ] Valid key returns 200 with features
- [ ] Revoked key returns valid: false
- [ ] Admin revoke requires auth
- [ ] Rate limit blocks excessive requests
