<objective>
Split server.js from 5,169 lines into 15 focused route modules under routes/, leaving server.js as a lean bootstrap file. No endpoint behavior changes — this is a pure structural refactor.

Purpose: server.js is the file most likely to cause merge conflicts and the hardest to navigate when debugging. Splitting it by domain makes each route group independently readable and sets up Phase 25 to add chat agent builtins to routes/chat.js without touching unrelated code.

Output: routes/ directory with 15 router files, server.js under 800 lines.

NOTE ON LINE TARGET: Analysis of server.js reveals the startup section (imports, SSL init, Express app creation, config loading, middleware, rate limiter registrations) spans ~620 lines (1-619) before any route handler. Rate limiters are app.use() middleware that stay in server.js because they run before router dispatch and share in-memory buckets. Extracting all 15 route groups frees ~4,400+ lines from server.js; with the retained startup section (~620 lines) plus retained routes (~180 lines), the realistic target is under 800 lines. Do not attempt to move rate limiters to individual routers — this would require passing limiters via appContext and changes the rate-limiting semantics.
</objective>
